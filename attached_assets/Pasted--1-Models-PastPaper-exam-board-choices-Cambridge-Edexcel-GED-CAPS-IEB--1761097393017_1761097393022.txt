
1. **Models**:
   - PastPaper:
       - exam_board (choices: Cambridge, Edexcel, GED, CAPS, IEB, ZIMSEC, Namibia Senior Secondary Certificate, Zambia Examinations Council, plus option to add new manually)
       - grade/level
       - subject
       - paper_type
       - paper_code (unique code from the exam board)
       - year
       - file upload (PDF/Word)
   - Quiz:
       - exam_board, grade, subject, topic
       - section (optional)  # e.g., Section A, Section B
       - chapter (optional)  # e.g., Cells, Genetics
       - google_form_link
       - is_premium (boolean)
       - ai_generated (boolean)
       - created_from_paper (ForeignKey to PastPaper)
   - Teacher: name, email, grade, subject
   - QuizResponse: quiz (ForeignKey), student_name, teacher_code, answers_json, score, submitted_at

2. **Admin Actions / Dashboard**:
   - **Question Bank Module**:
       - Upload and manage past papers (PDF/Word).  
       - **Bulk/folder upload** of multiple past papers at once.  
       - **Require input of metadata** for each uploaded paper:
           - Exam board (dropdown + manual entry)
           - Paper code
           - Subject, grade, year, paper type  
       - Organize the question bank by **Exam Board → Grade → Subject → Chapter → Section → Topic**.  
       - Generate AI quizzes from past papers or topics and store them in the Question Bank.
           - AI quizzes can be **chapter-wise and/or section-wise**.  
           - Admin can assign chapter/section when generating quizzes.  
       - Set quizzes as **free or premium**.  
       - Edit or retire outdated quizzes.  
   - **Teacher Code + Pre-Filled Google Form Links**:
       - For each teacher, generate a **unique code**.  
       - Automatically append this code to the Google Form link (pre-filled field).  
       - Display a note: "**Students must never share this link to avoid spam.**"  
       - Fetch responses from Google Forms via API and filter by teacher code so each teacher sees only their students’ submissions.
   - Manage teachers (add, edit, assign grades/subjects).  
   - View analytics:
       - Number of times each quiz is used  
       - Average student scores per quiz  
       - Analytics can also be **chapter/section-wise**

3. **Database**:
   - Single SQLite database for MVP
   - All models linked appropriately (PastPaper → Quiz → QuizResponse; Teachers linked where needed)

4. **UI Requirements**:
   - Responsive admin dashboard with sidebar or tabs: Question Bank, Past Papers, Quizzes, Teachers, Analytics
   - **Forms for bulk/folder upload**:
       - Admin must input metadata for each file (exam board, paper code, grade, subject, paper type, year)
   - Quiz list should show: title/topic, exam board, grade, subject, chapter, section, difficulty, AI-generated (yes/no), free/premium status, Google Form link
   - Pre-filled Google Form links displayed per teacher with warning note for students

5. **Integration**:
   - Google Forms API for creating quizzes and fetching responses
   - Placeholder for AI quiz generation function, including chapter-wise and section-wise categorization

6. **Generate**:
   - Django project structure with an **admin_panel app**
   - Models, views, templates, urls for admin dashboard
   - Sample HTML templates for:
       - Uploading past papers (single and bulk/folder upload with metadata + manual exam board entry)  
       - Managing the Question Bank  
       - Generating quizzes (chapter-wise and section-wise)  
       - Viewing analytics (including chapter/section performance)
